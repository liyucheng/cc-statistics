## 简介

**cc-statistics** 是一个用于统计 Claude Code 会话工程指标的 CLI 工具。

### 主要功能

- 📊 分析 AI 使用统计（指令数、工具调用、时长、代码行数、Token）
- 📅 时间范围过滤和报告生成
- 🔄 Git Hook 自动集成
- 🔍 项目对比和趋势分析
- 💰 费用预测和配额管理
- 📋 多种报告格式（终端、Markdown）

***

### 5 项核心指标

| 指标    | 说明             | 计算方式                           |
| ----- | -------------- | ------------------------------ |
| 指令数   | 用户发送给 AI 的指令数量 | type="user" 且非工具结果/元数据的消息      |
| 工具调用  | AI 调用工具的总次数    | 所有 type="tool" 的消息             |
| 活跃时长  | 实际编码时间         | 消息间隔 ≤5 分钟累加                   |
| 代码行数  | 新增/删除的代码行数     | Edit/Write 工具的 input 参数统计      |
| Token | 消耗的 Token 数量   | input\_tokens + output\_tokens |

## 基础使用

### 1. 查看帮助

```bash
# 查看完整帮助
cc-stats --help

# 查看命令概览
cc-stats
```

### 2. 列出所有项目

```bash
# 列出 Claude Code 识别的所有项目
cc-stats --list
```

输出示例：

```
Available projects:
  - myproject (12 sessions)
  - webapp (8 sessions)
  - cli-tool (5 sessions)
```

### 3. 分析单个项目

```bash
# 按项目名称匹配（支持模糊匹配）
cc-stats myproject

# 分析当前目录的项目（如果在 Claude Code 项目中）
cc-stats
```

### 4. 分析所有项目

```bash
# 分析所有项目并显示总览
cc-stats --all
```

### 5. 限制会话数量

```bash
# 只分析最近 5 个会话
cc-stats --all --last 5

# 只分析最近 10 个会话
cc-stats --all --last 10
```

## 时间过滤

### 1. 按时间范围分析

```bash
# 最近 7 天
cc-stats --all --since 7d

# 最近 2 周
cc-stats --all --since 2w

# 最近 1 个月
cc-stats --all --since 1m

# 指定日期范围
cc-stats --all --since 2026-04-01 --until 2026-04-15
```

### 2. 相对时间语法

| 语法           | 含义   | 示例           |
| ------------ | ---- | ------------ |
| `Nd`         | N 天前 | `7d` = 7 天前  |
| `Nw`         | N 周前 | `2w` = 2 周前  |
| `Nm`         | N 月前 | `1m` = 1 月前  |
| `YYYY-MM-DD` | 具体日期 | `2026-04-01` |

### 3. 组合使用

```bash
# 分析最近一周，限制 10 个会话
cc-stats --all --since 1w --last 10

# 分析指定项目的最近 3 天
cc-stats myproject --since 3d
```

## 报告生成

### 1. 生成周报

```bash
# 生成本周周报
cc-stats --report week

# 生成指定周的周报
cc-stats --report week --since 2026-04-01 --until 2026-04-07

# 保存到文件
cc-stats --report week -o weekly-report-2026-04.md
```

周报内容包括：

```markdown
# AI Coding 周报 (2026-04-01 至 2026-04-07)

## 总览
- 会话数：25
- 指令数：142
- 工具调用：893
- 活跃时长：8h 32m
- 代码行数：+2,847 / -512
- Token：284,321
- 费用：$1.42

## 每日统计
| 日期 | 会话 | 指令 | 时长 | 代码 |
|------|------|------|------|------|
| 04-01 | 4 | 23 | 1h 12m | +450/-80 |
| 04-02 | 3 | 18 | 45m | +380/-45 |
...

## 项目统计
| 项目 | 会话 | 指令 | 代码 |
|------|------|------|------|
| myproject | 12 | 68 | +1,234/-200 |
...
```

### 2. 生成月报

```bash
# 生成本月月报
cc-stats --report month

# 生成指定月的月报
cc-stats --report month --since 2026-04-01 --until 2026-04-30

# 保存到文件
cc-stats --report month -o monthly-report-2026-04.md
```

### 3. 自定义报告

```bash
# 指定时间范围
cc-stats --report week --since 2026-04-15 --until 2026-04-21 -o report.md

# 指定项目
cc-stats --report week myproject -o myproject-weekly.md
```

## Git Hook 集成

Git Hook 功能让你在每次 Git 提交时自动记录 AI 使用统计。

### 1. 安装 Git Hook

```bash
# 在项目目录下安装 post-commit hook
cc-stats --install-git-hook

# 安装 pre-commit hook（提交前记录）
cc-stats --install-git-hook --hook-type pre-commit

# 指定自定义日志文件
cc-stats --install-git-hook --log-file .logs/ai-usage.log
```

### 2. 使用流程

```bash
# 1. 正常开发（使用 Claude Code）
claude "帮我实现一个 RESTful API"

# 2. 提交代码（hook 自动记录统计）
git add .
git commit -m "Add RESTful API endpoints"

# 3. 查看自动生成的日志
cc-stats --read-log .ai-usage.log
```

### 3. 查看日志

```bash
# 读取并格式化日志
cc-stats --read-log .ai-usage.log

# 读取默认日志文件
cc-stats --read-log

# 查看原始日志
cat .ai-usage.log
```

日志内容示例：

```
---
时间：2026-04-18 14:32:15
提交人：didi <didi@example.com>
提交消息：Add RESTful API endpoints
分支：main

统计：
  指令数：8
  工具调用：45
  活跃时长：52m
  代码：+234 / -45
  Token：23,456
  费用：$0.12

模型分布：
  claude-sonnet-4: 100%

---
```

### 4. 生成 Hook 脚本

```bash
# 生成 hook 脚本到标准输出
cc-stats --generate-git-hook

# 保存到文件
cc-stats --generate-git-hook > .git/hooks/post-commit

# 赋予执行权限
chmod +x .git/hooks/post-commit
```

### 5. 手动写入日志

```bash
# 手动记录当前项目的统计到日志
cc-stats --write-log .ai-usage.log --repo /path/to/repo

# 不指定仓库路径则使用当前目录
cc-stats --write-log .ai-usage.log
```

### 6. 卸载 Git Hook

```bash
# 删除 post-commit hook
rm .git/hooks/post-commit

# 删除 pre-commit hook
rm .git/hooks/pre-commit
```

| Skill                | 调用次数 | 成功 | 失败 |

### 2. Git 集成分析

```bash
# 分析 Git 提交与 AI 使用的关系
cc-stats --git

# 显示每次提交的 AI 使用统计
cc-stats --git --all
```

### 3. Usage Quota 预测

```bash
# 预测未来使用情况
cc-stats --rate-limit

# 基于最近 N 天预测
cc-stats --rate-limit --since 7d

# 预测未来 30 天的使用
cc-stats --rate-limit --predict-days 30
```

输出示例：

```
Usage Quota 预测 (基于最近 7 天)

当前使用：
  Token/天：35,234
  费用/天：$0.18

预测（未来 30 天）：
  预计 Token：1,057,020
  预计费用：$5.29

建议：
  - 按当前速度，月底费用约为 $5.40
  - 建议关注高频使用的会话
```

### 4. 导出会话

```bash
# 导出匹配关键词的会话
cc-stats --export-chat "RESTful API"

# 导出到指定文件
cc-stats --export-chat "API" -o api-sessions.json

# 导出为 Markdown
cc-stats --export-chat "API" -o api-sessions.md
```

### 5. Webhook 通知

```bash
# 发送统计到 Webhook
cc-stats --notify https://hooks.slack.com/services/xxx

# 发送指定项目的统计
cc-stats myproject --notify https://hooks.slack.com/services/xxx

# 发送周报
cc-stats --report week --notify https://hooks.slack.com/services/xxx
```

### 6. 安装 Claude Code Hooks

```bash
# 安装到 Claude Code 全局配置
cc-stats --install-hooks

# 这会修改 ~/.claude/settings.json
```

配置内容：

```json
{
  "hooks": {
    "onSessionComplete": "cc-stats-hooks --on-session-complete",
    "onPermissionRequest": "cc-stats-hooks --on-permission-request",
    "onRateLimit": "cc-stats-hooks --on-rate-limit"
  }
}
```

## 实际使用场景

### 场景 1：每日工作总结

```bash
# 每天下班前运行，查看今天的 AI 使用
cc-stats --all --since 1d

# 生成日报
cc-stats --report week --since $(date -v-1d +%Y-%m-%d) -o daily-$(date +%Y-%m-%d).md
```

### 场景 2：周报生成

```bash
# 每周五生成周报
cc-stats --report week -o weekly-$(date +%Y-%m%d).md

# 发送到 Slack
cc-stats --report week --notify https://hooks.slack.com/services/xxx
```

### 场景 3：项目成本分析

```bash
# 分析项目的 AI 使用成本
cc-stats myproject --since 1m

# 对比多个项目的成本
cc-stats --compare --since 1m
```

### 场景 4：Git 提交时的自动记录

```bash
# 初始化项目时安装 Git Hook
git init
cc-stats --install-git-hook

# 之后每次提交都会自动记录
git add .
git commit -m "..."  # 自动记录 AI 使用统计

# 查看提交历史和 AI 使用
git log --oneline
cc-stats --read-log .ai-usage.log
```

### 场景 5：团队协作

```bash
# 每个人安装 Git Hook
cc-stats --install-git-hook --log-file .logs/team-usage.log

# 提交后自动记录

# 汇总团队使用
cc-stats --read-log .logs/team-usage.log
```

### 场景 6：监控和预警

```bash

# 设置定时任务（cron）
# 每天早上 9 点发送周报
0 9 * * 1 cc-stats --report week --notify https://hooks.slack.com/services/xxx
```

### 场景 7：性能优化

```bash

# 分析工具调用模式

```

## 常见问题

### Q: 如何统计所有时间的数据？

A:

```bash
# 不指定时间范围
cc-stats --all
```

### Q: 如何查看某个会话的详细信息？

A:

```bash
# 导出特定会话
cc-stats --export-chat <keyword>

# 导出为 JSON 或 Markdown
cc-stats --export-chat <keyword> -o session.md
```

### Q: Git Hook 不工作？

A:

```bash
# 检查 hook 是否存在
ls -la .git/hooks/post-commit

# 检查权限
chmod +x .git/hooks/post-commit

# 手动测试
.git/hooks/post-commit
```

### Q: 如何统计多个项目？

A:

```bash
# 使用 --all
cc-stats --all

# 或使用模式匹配
cc-stats --pattern "myproject|webapp"
```

### Q: 如何合并多个日志文件？

A:

```bash
# 手动合并
cat .logs/*.log > combined.log

# 或使用脚本
for log in .logs/*.log; do
  cat "$log" >> combined.log
done

# 然后读取合并后的日志
cc-stats --read-log combined.log
```
