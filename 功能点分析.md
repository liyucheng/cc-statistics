# cc-statistics 功能点分析

## 项目概述

cc-statistics 是一个统一的 AI coding 会话统计工具，支持 **Claude Code / Gemini CLI / Codex CLI / Cursor** 四个平台。纯 Python 标准库实现，零第三方依赖，100% 本地运行。

---

## 核心功能模块

### 1. 会话解析 (parser.py)

**核心类：**
- `ToolCall` - 工具调用记录（名称、输入、时间戳、工具 ID）
- `Message` - 消息记录（角色、内容、时间戳、token 使用量、工具调用列表）
- `Session` - 会话记录（会话 ID、项目路径、源平台、消息列表）

**主要功能：**

#### 1.1 Claude Code 会话解析
- `parse_jsonl()` - 解析 Claude Code 的 JSONL 格式会话文件
- 流式去重机制：按 message_id 去重 assistant 消息，保留 output_tokens 最大的记录
- 提取工具调用 (tool_use) 和工具返回 (tool_result)
- 识别用户消息：`type == "user"` 且 `is_tool_result == False` 且 `is_meta == False`

#### 1.2 Codex 会话解析
- `parse_codex_jsonl()` - 解析 OpenAI Codex rollout JSONL 格式
- 工具名称映射：Codex 工具名 → cc-stats 统一名称（如 `exec_command` → `Bash`）
- Token 使用量转换：Codex 的 input_tokens 包含 cached_input_tokens，转换为统一口径
- 模型信息提取：从 payload 中提取 model 或 collaboration_mode.settings.model
- 消息去重：使用 `(timestamp, text)` 作为 key 去重

#### 1.3 Gemini CLI 会话解析
- `parse_gemini_jsonl()` - 解析 Gemini CLI 的 JSONL 格式
- 兼容 Gemini 特有的消息结构
- 支持 Gemini 2.5 Pro/Flash 模型定价

#### 1.4 会话文件发现
- `find_sessions()` - 查找 `~/.claude/projects/` 下所有 JSONL 文件
- `find_sessions_by_keyword()` - 按关键词匹配项目（在目录名和 cwd 中搜索）
- `find_codex_sessions()` / `find_codex_sessions_by_keyword()` - Codex 会话查找
- `find_gemini_sessions()` / `find_gemini_sessions_by_keyword()` - Gemini 会话查找
- 防御性过滤：跳过 `agent-*` 和 `subagents/` 子目录下的文件，避免双重计数

#### 1.5 统一解析入口
- `parse_session_file()` - 根据文件类型自动选择解析器

### 2. 统计分析 (analyzer.py)

**核心类：**
- `SessionStats` - 会话统计数据（指令数、工具调用、时长、代码行数、token、Git 集成数据）
- `TokenUsage` - Token 使用量（input/output/cache_read/cache_creation）

**主要功能：**

#### 2.1 会话分析
- `analyze_session()` - 从 Session 计算 5 项工程指标
  - **指令数**：非 tool_result 且非 meta 的 user 消息数量
  - **工具调用**：assistant 消息中的 tool_use 调用次数
  - **活跃时长**：消息间隔 ≤ 5 分钟的累积时长
  - **代码行数**：从 Edit/Write 工具调用的 input 参数计算新增/删除行数
  - **Token 消耗**：按日期分组统计（token_by_date）和按模型分组统计（token_by_model）

#### 2.2 时间分析
- `active_duration` - 用户活跃时间（相邻消息间隔 ≤ 5 分钟）
- `ai_duration` - AI 处理时间（assistant 消息的 timestamp 累积）
- `user_duration` - 用户等待时间（user 消息到下一个 assistant 消息的间隔）

#### 2.3 代码变更统计
- 从 `Edit`/`Write` 工具调用的 input 参数提取
- 按语言分组统计：`lines_by_lang`
- `total_added` / `total_removed` - 总计新增/删除行数

#### 2.4 工具调用分析
- `tool_call_total` - 总调用次数
- `tool_call_counts` - 按工具名分组统计
- 区分 Skill 和 MCP 工具

#### 2.5 Token 使用统计
- `token_usage` - 总计 Token 使用量
- `token_by_date` - 按日期分组（自然键：YYYY-MM-DD）
- `token_by_model` - 按模型分组

#### 2.6 Git 集成统计
- `git_commit_count` - 期间提交次数
- `git_ai_commit_count` - AI 生成的提交次数（commit message 包含 "AI" 或 "claude"）
- `git_total_added` / `git_total_removed` - Git 代码变更统计

#### 2.7 效率评分
- `efficiencyGrade` - 综合评分（S/A/B/C/D，0-100）
- 维度：
  - 代码产出率 (40分)：code_per_1k_token，0.5 行/K token 以上满分
  - 指令精准度 (30分)：avg_tokens_per_msg，50K 以下满分
  - AI 利用率 (30分)：ai_duration / active_duration，70% 以上满分

#### 2.8 统计合并
- `merge_stats()` - 合并多个 SessionStats

### 3. 报告生成 (reporter.py)

**主要功能：**

#### 3.1 Markdown 报告生成
- `generate_report(period)` - 生成周报/月报
  - **概览表格**：会话数、指令数、工具调用、活跃时长、AI 处理时间、用户活跃时间、Token 消耗、预估费用、代码新增/删除、Git Commits
  - **每日明细表格**：日期、会话数、指令数、活跃时长、Token、费用
  - **工具调用 Top 5**：工具名和调用次数
  - **代码变更（按语言）**：语言、新增、删除、净增
  - **Token 消耗（按模型）**：模型、Token 数、费用
  - **项目分布**：项目名、会话数、指令数、费用（Top 10）

#### 3.2 周期对比
- 与上一周期的数据对比
- 计算变化百分比（会话数、指令数、Token、费用、代码新增）

#### 3.3 成本预测
- 日均费用、月度预测
- 活跃天数统计

#### 3.4 跨日 Session 处理
- 跨日 session 的 token 分配到各自然日
- 费用按当天 token 占该 session 总 token 的比例估算

### 4. Git 集成 (git_integration.py)

**核心类：**
- `CommitInfo` - Git commit 信息（hash、时间戳、作者、消息、代码变更统计）
- `CommitCost` - commit 归属的 AI 会话成本
- `GitIntegrationResult` - 完整分析结果

**主要功能：**

#### 4.1 Git 日志解析
- `parse_git_log()` - 通过 `git log --numstat` 解析 commit 列表
  - 提取 commit hash、时间戳、作者、消息
  - 计算每个 commit 的代码新增/删除行数
  - 支持 `--since` / `--until` 时间过滤

#### 4.2 会话归属算法
- `attribute_sessions_to_commits()` - 将 session 按时间归属到 commit
  - **归属规则**：
    - commit 时间窗口 = (上一个 commit 时间, 当前 commit 时间]
    - 第一个 commit 的窗口 = (commit_time - 24h, commit_time]
    - 如果 session 时间范围与窗口有交集，按交集占 session 总时长的比例分配 token
  - 支持零时长 session：只检查点是否在窗口内

#### 4.3 成本计算
- `_estimate_cost()` - 估算 token 费用（使用Claude 模型定价）
- 统计每个 commit 的：
  - 归属的 session 数
  - Token 使用量（input/output/cache_read）
  - 预估费用

#### 4.4 完整分析
- `analyze_git_integration()` - 执行完整的 Git 集成分析
  - 解析 git log
  - 提取 session 摘要
  - 归属 session 到 commit
  - 汇总统计

### 5. 通知系统 (notifier.py)

**配置文件：** `~/.cc-stats/notify_config.json`

**主要功能：**

#### 5.1 通知发送策略
1. 优先使用 UserNotifications（通过 Swift App 中转）
2. 降级使用 osascript (AppleScript) 作为 fallback
3. 可配置 webhook 转发到飞书/钉钉/Slack/Discord

#### 5.2 智能抑制
- 终端窗口有焦点时不弹通知（避免打扰正在看的用户）
- 可通过配置文件关闭指定类型的通知
- 检测当前 frontmost 应用的 bundle ID，与已知终端 bundle ID 列表比较

#### 5.3 高级通知函数
- `notify_session_complete()` - 会话完成通知
  - 显示项目名、时⻓、Token 数、费用
- `notify_cost_alert()` - 费用预警通知
  - 单日/每周费用超过上限时触发
  - 使用 Sosumi 声音，强制通知（不抑制）
- `notify_permission_request()` - 权限请求通知
  - Claude Code 需要用户确认时提醒
  - 显示工具名和描述（截断过长描述）

#### 5.4 通知类型控制
- `session_complete` - 会话完成通知
- `cost_alert` - 费用预警通知
- `permission_request` - 权限请求通知
- `general` - 通用通知

#### 5.5 Webhook 转发
- 支持自动检测平台：
  - 飞书（feishu.cn / larksuite.com）
  - 钉钉（dingtalk.com）
  - Slack（hooks.slack.com）
  - Discord（discord.com）
- 平台特定的消息格式适配

### 6. Hook 系统 (hooks.py)

**支持的 Hook 事件：**
- `Stop` - 会话结束时发送完成通知
- `PreToolUse` - 需要权限确认时发送通知
- `Notification` - Claude Code 空闲等待用户输入时通知

**主要功能：**

#### 6.1 Hook 事件处理
- `handle_stop()` - 处理 Stop 事件
  - 从 stop_reason 判断是否正常结束（用户取消不通知）
  - 快速统计当前会话（轻量级，不做完整解析）
  - 发送会话完成通知

- `handle_pre_tool_use()` - 处理 PreToolUse 事件
  - 提取工具名和描述
  - 发送权限请求通知

- `handle_notification()` - 处理 Notification 事件
  - idle_prompt 类型：发送等待输入通知

#### 6.2 Hook 安装/卸载
- `install_hooks(scope)` - 安装 Claude Code hooks
  - scope: "user" (全局) 或 "project" (当前项目)
  - 自动写入 `~/.claude/settings.json` 或 `.claude/settings.local.json`
  - 避免重复安装

- `uninstall_hooks(scope)` - 卸载 Claude Code hooks
  - 清理对应的 hook 配置
  - 兼容旧格式和新格式

#### 6.3 Hook 命令获取
- `get_hook_command()` - 获取当前环境的 hook 命令
  - 优先查找 cc-stats-hooks entry-point binary（uv/pipx 安装场景）
  - 找不到时 fallback 到 `python -m cc_stats.hooks`

### 7. 导出功能 (exporter.py)

**主要功能：**

#### 7.1 会话导出
- `export_session()` - 将会话导出为 Markdown
  - 包含标题、时间、项目路径、会话 ID
  - 格式化用户和 assistant 消息
  - 可选包含工具调用（默认跳过纯工具调用）

#### 7.2 查找并导出
- `find_and_export(keyword, output, include_tools)` - 查找会话并导出
  - 先按 session ID 前缀匹配
  - 再按内容搜索（在所有消息的文本中搜索）
  - 支持输出到文件或 stdout

### 8. 定价系统 (pricing.py)

**模型定价数据：**
- Claude Opus / Sonnet / Haiku 系列
- OpenAI GPT-4o / GPT-5 系列 / Codex 系列
- Gemini 2.5 Pro / Flash / Flash Lite 系列

**主要功能：**

#### 8.1 定价匹配
- `match_model_pricing()` - 根据模型名匹配单价
  - 支持模糊匹配（如包含 "sonnet" 匹配所有 Sonnet 版本）
  - 厂商回退策略：未知模型按同厂商主力模型保守回退
  - 防止历史脏数据导致费用完全丢失

#### 8.2 费用估算
- `estimate_cost_from_token_by_model()` - 按 token_by_model 估算总费用
  - 计算方式：input/output/cache_read/cache_create 各自计费后求和

#### 8.3 价格标准
- 所有价格为 USD per 1M tokens
- 数据来源：
  - OpenAI: https://developers.openai.com/api/docs/pricing
  - Anthropic: https://platform.claude.com/docs/en/about-claude/pricing
  - Gemini: https://ai.google.dev/gemini-api/docs/pricing

### 9. CLI 入口 (cli.py)

**命令：** `cc-stats`

**主要功能：**

#### 9.1 基本命令
- `cc-stats` - 分析当前目录项目的所有会话
- `cc-stats <path>` - 分析指定 JSONL 文件或项目目录
- `cc-stats <keyword>` - 按关键词匹配项目
- `cc-stats --all` - 分析所有项目的所有会话
- `cc-stats --list` - 列出所有已知项目

#### 9.2 过滤选项
- `--last N` - 只分析最近 N 个会话
- `--since TIME` - 只包含此时间之后的会话
  - 支持格式：2026-03-13, 2026-03-13T10:00, 3d, 2w, 1h
- `--until TIME` - 只包含此时间之前的会话

#### 9.3 报告生成
- `--report week` - 生成周报 Markdown
- `--report month` - 生成月报 Markdown
- `--report week --since 2026-03-01 --until 2026-03-07 -o report.md` - 自定义时间范围

#### 9.4 高级功能
- `--compare` - 对比所有项目的关键指标
  - 按 Token 总量降序排列
  - 显示：会话数、指令数、活跃时长、Token、费用、代码变更
- `--skills` - 展示 Skill 使用统计
  - 调用次数、成功率、时间分布
- `--export <keyword> -o output.md` - 导出会话为 Markdown
- `--git <repo-path>` - 显示 Git 集成分析
  - 将会话按时间归属到 commit，计算每 commit 的 AI 成本
- `--rate-limit` - 显示 Usage Quota 预测
  - 分析最近 1 小时内的所有会话
  - 显示预估剩余时间、预测重置时间、风险等级

#### 9.5 Hook 管理
- `--install-hooks` - 安装 Claude Code hooks
- `--install-hooks --scope project` - 安装到当前项目
- `--uninstall-hooks` - 卸载 Claude Code hooks

#### 9.6 版本检查
- `--check-update` - 检查更新
- `--self-update` - 自我更新（仅 pipx 安装）
- 启动时自动显示更新提示（基于缓存）

#### 9.7 测试功能
- `--notify test` - 测试通知
- `--notify session` - 测试会话完成通知
- `--notify cost` - 测试费用预警通知
- `--notify permission` - 测试权限请求通知

### 10. 格式化输出 (formatter.py)

**主要功能：**

#### 10.1 统计表格格式化
- `format_stats()` - 格式化 SessionStats 为终端表格
  - 项目信息、会话统计、时间分析、代码变更、Token 使用、Git 集成
  - 支持多平台合并显示

#### 10.2 工具调用表格
- 格式化工具调用 Top 10
- 区分 Skill 和 MCP 工具

#### 10.3 Skill 统计
- `format_skill_stats()` - 格式化 Skill 使用统计
  - 调用次数、成功率、错误次数、平均耗时

#### 10.4 Git 集成格式化
- `format_git_integration()` - 格式化 Git 集成结果
  - commit 信息、归属的 session 数、Token、费用
  - 按费用降序排列

#### 10.5 Rate Limit 格式化
- `format_rate_limit()` - 格式化 Usage Quota 预测
  - 当前消耗、预估剩余时间、重置时间、风险等级
  - 风险等级：Low (绿色) / Medium (黄色) / High (红色)

- `_fmt_duration()` - 格式化时长（如 "1h 23m 45s"）

### 11. 版本检查 (version_checker.py)

**主要功能：**

#### 11.1 版本检查
- `check_for_update()` - 检查 GitHub 上的最新版本
  - 从 PyPI 获取版本信息（无 API key，公开 API）
  - 缓存结果到 `~/.cc-stats/update_cache.json`
  - 缓存有效期：24 小时

#### 11.2 缓存读取
- `get_cached_update()` - 读取缓存的更新信息
  - 返回 None 如果缓存不存在或过期

#### 11.3 消息格式化
- `format_update_message()` - 格式化更新提示
  - 显示当前版本、最新版本、更新命令

#### 11.4 自我更新
- `self_update()` - 使用 pipx 自我更新
  - 仅限 pipx 安装场景

### 12. Rate Limit 分析 (rate_limiter.py)

**主要功能：**

#### 12.1 Usage Quota 预测
- `analyze_rate_limit()` - 分析最近 1 小时内的 token 消耗
  - 计算当前消费速率
  - 预估何时会达到配额上限
  - 计算风险等级（Low/Medium/High）

#### 12.2 配额限制
- 支持 Claude Anthropic API 的 usage quota 限制
- 可配置窗口限制（如 100 万 tokens / 天）

#### 12.3 时间窗口计算
- 根据当前 token 消耗速率和配额限制
- 计算剩余时间、重置时间
- 支持不同重置周期（按小时/天）

### 13. 其他功能

#### 13.1 Unicode 处理
- CJK 字符宽度计算（用于终端表格对齐）
- 终端显示宽度计算：`_display_width()` / `_pad_right()` / `_pad_left()``

#### 13.2 时间解析
- `_parse_time_arg()` - 解析时间参数
  - 绝对时间：2026-03-13, 2026-03-13T10:00
  - 相对时间：1h, 3d, 2w
  - 支持时区转换（本地时间 → UTC）

#### 13.3 项目名称解析
- `_resolve_project_name()` - 从 JSONL 文件中的 cwd 字段还原项目真实路径
- fallback: 目录名本身

#### 13.4 日期范围裁剪
- `_trim_stats_by_date_range()` - 按本地日期范围裁剪 token_by_date
  - 跨越日期范围的 session 只计入范围内日期的 token
  - 从裁剪后的 token_by_date 重算 token_usage 总量

## 数据流架构

```
Claude Code / Codex / Gemini CLI 会话文件 (JSONL)
        ↓
parser.py - 会话解析
        ↓
analyzer.py - 统计分析 (5 项工程指标 + 效率评分)
        ↓
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│         │         │         │         │         │
formatter  reporter  git_integration  notifier  exporter
(表格输出)  (周报/月报)  (Git 集成)  (系统通知)  (会话导出)
```

## 技术特性

### 1. 零依赖
- 纯 Python 标准库实现
- 无第三方依赖（requests/pandas/等）

### 2. 跨平台
- macOS / Linux / Windows
- 平台特定的功能通过条件编译实现

### 3. 性能优化
- 流式解析 JSONL（不一次性加载整个文件）
- 防御性过滤（避免重复计数）
