## 二、新增功能

### Git Hook 集成

在每次 Git 提交时自动记录 AI 使用统计到日志文件。

#### 安装 Git Hook

```bash
# 在你的项目目录下
cc-stats --install-git-hook

# 指定自定义日志文件
cc-stats --install-git-hook --log-file .logs/ai-usage.log
```

#### 使用示例

```bash
# 1. 正常开发（使用 Claude Code）
claude "帮我实现一个 RESTful API"

# 2. 提交代码（hook 自动记录统计）
git add .
git commit -m "Add RESTful API endpoints"

# 3. 查看统计日志
cc-stats --read-log .ai-usage.log
```

#### 日志内容

每次提交记录：
- 指令数（用户消息数量）
- 工具调用次数
- 活跃时长
- 代码新增/删除行数
- Token 消耗
- 预估费用
- 提交人信息（用户名、邮箱、提交消息）
- 模型分布

## 三、常用命令

### 基础分析

```bash
# 分析当前目录项目
cc-stats

# 分析指定项目
cc-stats myproject

# 列出所有可用项目
cc-stats --list

# 分析所有项目
cc-stats --all
```

### 时间过滤

```bash
# 最近 7 天
cc-stats --all --since 7d

# 最近 2 周
cc-stats --all --since 2w

# 指定日期范围
cc-stats --all --since 2026-04-01 --until 2026-04-15
```

### 报告生成

```bash
# 生成周报
cc-stats --report week

# 生成月报
cc-stats --report month

# 自定义时间范围
cc-stats --report week --since 2026-04-01 --until 2026-04-07 -o report.md
```

### Git Hook 操作

```bash
# 安装 Git Hook
cc-stats --install-git-hook

# 指定日志文件
cc-stats --install-git-hook --log-file .logs/ai-usage.log

# 使用 pre-commit hook
cc-stats --install-git-hook --hook-type pre-commit

# 生成 hook 脚本
cc-stats --generate-git-hook > .git/hooks/post-commit

# 手动写入日志
cc-stats --write-log .ai-usage.log --repo /path/to/repo

# 读取并格式化日志
cc-stats --read-log .ai-usage.log
```

### 高级功能

```bash
# 项目对比
cc-stats --compare --since 1w

# Skill 使用统计
cc-stats --skills

# Git 集成分析
cc-stats --git

# Usage Quota 预测
cc-stats --rate-limit

# 导出会话
cc-stats --export-chat <keyword>

# Webhook 通知
cc-stats --notify https://hooks.slack.com/services/xxx

# 安装 Claude Code Hooks
cc-stats --install-hooks
```

## 九、开发模式 vs 生产模式

### 开发模式（当前使用）

```bash
# 从本地源码安装
uv tool install .
```

优点：
- 代码修改后只需重新安装
- 无需发布到 PyPI
- 适合测试和开发

缺点：
- 每次修改需要重新安装
- 仅本机可用

### 生产模式

```bash
# 从 PyPI 安装
uv tool install cc-statistics
```

优点：
- 一键安装
- 可以在任何机器使用
- 版本稳定

缺点：
- 需要发布到 PyPI
- 修改代码需要重新发布

## 十、发布到 PyPI（可选）

如果你想发布修改版到 PyPI：

### 1. 更新版本号

编辑 `pyproject.toml`：

```toml
[project]
name = "cc-statistics"
version = "0.13.0"  # 修改版本号
```

### 2. 构建

```bash
pip install build twine
python -m build
```

### 3. 发布到 TestPyPI（测试）

```bash
twine upload --repository testpypi dist/*
```

### 4. 发布到 PyPI

```bash
twine upload dist/*
```

### 5. 使用

```bash
# 安装发布版
uv tool install cc-statistics

# 或指定版本
uv tool install cc-statistics==0.13.0
```
